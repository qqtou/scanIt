# ScanIt / 扫客 — 概要设计

## 1. 系统架构

采用 **插件化微服务架构**，核心服务 + 可插拔检测引擎，便于扩展新搜索引擎和新的比对算法。

```
┌─────────────────────────────────────────────────┐
│            Web 管理后台 / API 网关              │
└──────────────┬──────────────────────────────────┘
               │
    ┌──────────┴──────────┐
    │                     │
┌───▼────┐          ┌────▼────┐
│ 任务   │          │ 报告     │
│ 调度器 │          │ 生成器   │
└───┬────┘          └────┬────┘
    │                     │
    ▼                     ▼
┌─────────────────────────────┐
│      检索与比对引擎          │
│  ┌─────────┐  ┌─────────┐ │
│  │ 搜索引擎 │  │ 内容    │ │
│  │ 适配器   │  │ 比对器  │ │
│  └─────────┘  └─────────┘ │
└─────────────────────────────┘
               │
    ┌──────────┼──────────┐
    ▼          ▼          ▼
┌─────┐    ┌─────┐    ┌─────┐
│文本 │    │图片 │    │视频 │
│比对 │    │比对 │    │比对 │
│引擎 │    │引擎 │    │引擎 │
└─────┘    └─────┘    └─────┘
```

## 2. 模块划分

### 2.1 任务调度模块（scheduler）
- 定时触发检测任务（Cron / 间隔）
- 任务队列管理（Redis / RabbitMQ）
- 失败重试、断点续跑

### 2.2 检索引擎模块（search-engine）
- 搜索引擎适配器接口（Adapter Pattern）
- 已实现适配器：Google / Bing / 百度
- 负责关键字检索、结果解析、去重

### 2.3 内容比对模块（comparison-engine）
- **文本比对器**：基于 SimHash + LSH（局部敏感哈希）做大规模相似度检测；长文本用滑动窗口
- **图片比对器**：感知哈希（pHash / dHash）+ CNN 特征（ResNet / ViT）做特征匹配；部分匹配用 SIFT / ORB 做局部特征对齐
- **视频比对器**：关键帧提取（FFmpeg）→ 图片比对；音频指纹（Chromaprint）可选

### 2.4 报告生成模块（reporter）
- 模板化报告（Jinja2 / React PDF）
- 支持 HTML / PDF 导出
- 风险评分与排序

### 2.5 告警模块（alerter）
- 邮件告警（SMTP）
- Webhook 推送（企业微信 / 钉钉 / Slack）
- 告警等级分级

## 3. 技术选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 后端框架 | Python 3.11+ / FastAPI | 异步支持好，AI 库生态丰富 |
| 任务队列 | Celery + Redis | 成熟稳定，支持定时任务 |
| 检索引擎 | 自研适配器 | 封装各搜索引擎 API |
| 文本比对 | SimHash + LSH / Sentence-BERT | 大规模用 SimHash；高精度用 BERT |
| 图片比对 | OpenCV + PyTorch + timm | pHash 快速初筛；CNN 精细比对 |
| 视频处理 | FFmpeg + OpenCV | 关键帧提取；音频指纹用 Chromaprint |
| 存储 | PostgreSQL + Qdrant（向量库） | 关系数据用 PG；特征向量用 Qdrant |
| 前端 | React + Ant Design Pro | 管理后台 |
| 部署 | Docker Compose / K8s | 容器化部署 |

## 4. 数据流

```
用户输入（作品 + 关键字）
    ↓
任务调度器（创建检测任务）
    ↓
检索引擎（调用搜索引擎 API）
    ↓
原始结果（URL / 文本 / 图片 / 视频）
    ↓
内容下载器（抓取网页 / 图片 / 视频）
    ↓
特征提取（文本 → SimHash；图片 → pHash + CNN；视频 → 关键帧 → 图片特征）
    ↓
相似度比对（与客户作品特征比对）
    ↓
侵权判定（相似度 > 阈值 → 疑似侵权）
    ↓
报告生成器（汇总结果 → HTML / PDF）
    ↓
告警模块（高危侵权 → 实时告警）
```

## 5. 接口设计（概要）

### 5.1 内部模块接口（概要）

```python
# 搜索引擎适配器接口
class SearchEngineAdapter(ABC):
    @abstractmethod
    def search(self, query: str, num_results: int) -> List[SearchResult]:
        pass

# 内容比对器接口
class ContentComparator(ABC):
    @abstractmethod
    def compare(self, source: Content, target: Content) -> ComparisonResult:
        pass
```

### 5.2 外部 API（概要）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/tasks` | POST | 创建检测任务 |
| `/api/tasks/{id}` | GET | 查询任务状态 |
| `/api/reports/{task_id}` | GET | 获取检测报告 |
| `/api/works` | POST | 上传客户作品 |

## 6. 数据库设计（概要）

### works 表（客户作品）
- id, user_id, title, content_type, content_url, content_hash, created_at

### tasks 表（检测任务）

### results 表（检测结果）

## 8. 风险与应对

| 风险 | 应对措施 |
|------|---------|
| 搜索引擎 API 限流 | 多账号轮询；代理池；降级到网页抓取 |
| 误报率高 | 人工标注反馈循环；模型持续训练 |
| 视频比对耗时 | 分布式处理；GPU 加速；采样策略（降低帧率） |
| 客户作品泄露 | 加密存储；访问控制；审计日志 |
