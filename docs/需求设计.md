# ScanIt / 扫客 — 需求设计

## 1. 项目背景

客户拥有原创作品（文字、图片、视频等），需要监测互联网上是否存在未经授权的使用行为。传统人工监测效率低、覆盖面窄，亟需一套自动化侵权检测系统，通过搜索引擎检索 + 多媒体内容比对，快速发现潜在侵权内容。

## 2. 目标与范围

### 目标
- 通过关键字在搜索引擎 / 搜索服务商检索相关内容
- 对检索结果中的文本、图片、视频进行侵权比对
- 输出侵权风险报告，辅助人工审核

### 范围
- **纳入范围**：文本、图片、视频三类内容的侵权检测；支持主流搜索引擎接入
- **排除范围**：音频侵权检测（后期扩展）；实时流式监测（本期做定时批量检测）

## 3. 功能需求

### 3.1 搜索引擎检索模块
- 支持配置多个搜索引擎 / 搜索服务商（如 Google、Bing、百度等）
- 通过关键字检索，获取搜索结果（标题、URL、摘要、缩略图）
- 支持分页抓取，去重

### 3.2 文本侵权检测模块
- 输入：客户文本作品 + 搜索结果中的文本片段 / 网页全文
- 检测能力：
  - 大篇幅重复检测（相似度阈值可配置）
  - 段落 / 句子级别匹配
  - 改写 /  paraphrasing 检测（可选，后期扩展）
- 输出：相似度评分 + 重复片段标注

### 3.3 图片侵权检测模块
- 输入：客户图片作品 + 搜索结果中的图片
- 检测能力：
  - 完整图片相似度比对（感知哈希 / CNN 特征）
  - 部分区域匹配（检测作品的一部分是否出现在其他图片中）
  - 旋转 / 裁剪 / 滤镜鲁棒性
- 输出：相似度评分 + 匹配区域标注

### 3.4 视频侵权检测模块
- 输入：客户视频作品 + 搜索结果中的视频
- 检测能力：
  - 关键帧提取 + 图片相似度比对
  - 片段级重复检测（时间轴对齐）
  - 音频指纹比对（可选，后期扩展）
- 输出：相似度评分 + 重复时间段标注

### 3.5 报告与告警
- 生成侵权检测报告（PDF / HTML）
- 按风险等级排序（高 / 中 / 低）
- 支持邮件 / Webhook 告警

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 单次检测任务：文本 < 5s / 篇；图片 < 10s / 张；视频 < 60s / 5min 片段 |
| 准确性 | 图片 / 视频 top-10 结果准确率 > 85%（人工标注验证集） |
| 可扩展性 | 支持新增搜索引擎、新增比对算法插件化接入 |
| 可靠性 | 搜索接口失败重试；任务断点续跑 |
| 安全性 | 客户作品不对外泄露；API Key 加密存储 |

## 5. 用户角色与用例

### 用户角色
- **管理员**：配置搜索引擎、管理客户作品、查看报告
- **审核员**：查看侵权报告、人工复核、标记误报 / 确报
- **系统**：定时执行检测任务、发送告警

### 核心用例
1. 管理员上传客户作品（文本 / 图片 / 视频）
2. 管理员配置检测任务（关键字、搜索引擎、检测范围）
3. 系统定时执行检测，生成初步报告
4. 审核员查看报告，人工复核，标记结果
5. 系统发送高危侵权告警

## 6. 后续扩展方向

- 音频侵权检测
- 实时监测（流式）
- 改写 / paraphrasing 检测（NLP 高级模型）
- 区块链存证集成
- 自动化下架函生成与发送
